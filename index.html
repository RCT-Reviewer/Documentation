RCT-Reviewer is a modernized, standalone version of RobotReviewer, designed as a third-party reference tool for Risk of Bias assessment. It builds upon RobotReviewer's original machine learning models trained on 12,808 randomized controlled trials (RCTs).

Why Use RCT-Reviewer?

RCT-Reviewer is designed as a third-party tiebreaker reference for systematic reviews. Standard guidelines require two independent human reviewers; when they disagree, this tool provides an instant, objective, and data-driven third opinion to resolve ties.

  1. 1

    Near-Human Accuracy

    The system achieves 71.0% accuracy for Risk of Bias judgments, performing within <8% of human expert consensus (which stands at 78.3%) [1].

  2. 2

    Highly Precise Extraction

    In a randomized Cochrane user trial, the models demonstrated 87% Precision and 90% Recall for identifying the exact text snippets supporting the bias judgment [2].

  3. 3

    Validated Acceptance

    Real-world feasibility studies show that human reviewers accept the tool's judgments at a rate equal to that of their human peers (Risk Ratio 1.02) [3].

  4. 4

    Rigorous Methodology

    Developed by Marshall, Kuiper, and Wallace, the models were trained on 12,808 clinical trial PDFs using "distant supervision" to ensure high-quality classification without prohibitive manual labeling costs [1,4].

Architecture & Models

This project preserves the machine learning core of the original RobotReviewer while modernizing the infrastructure.

# The ML Pipeline
1. PDF Parsing:     PyMuPDF (fitz) + spaCy sentence segmentation
                   (No Java/GROBID dependency required)

2. Feature Extract: HashingVectorizer (scikit-learn)
                   converts text into high-dimensional sparse matrices

3. Classification:  MiniClassifier (lightweight Linear SVM wrapper)
                   loads pre-trained .npz weights

# SVM-Only Pipeline:
# CNN models removed due to TensorFlow compatibility issues
# on Python 3.11-3.12. Not required for accurate predictions.

Model Redistribution Note: This repository contains joblib-converted model artifacts originally developed in RobotReviewer. The old pickle model files were compressed and redistributed into .joblib format using convert_models.py for better compatibility and smaller file sizes.

Quick Start — Running RCT-Reviewer

Clone, install, and run the recommended mode (Hugging Face Hub):

# Clone the repository
git clone https://github.com/aurumz-rgb/RCT-Reviewer.git
cd RCT-Reviewer

# Create and activate virtual environment
python3.12 -m venv .venv
source .venv/bin/activate        # Linux/macOS
.venv\Scripts\activate           # Windows

# Install dependencies
pip install -r requirements.txt
python -m spacy download en_core_web_sm

# Run app2.py — Hugging Face Hub mode (recommended, no LFS required)
python -m streamlit run rct_reviewer/app2.py

Other modes: app.py uses local .joblib files (requires git lfs pull); app1.py uses legacy .pickle/.pck files (requires git lfs pull).

Note on SVM vs CNN

This project uses a Linear SVM-only pipeline instead of the original SVM + CNN ensemble. CNN models were removed because they depend on TensorFlow/Keras .h5 files which break on Python 3.11–3.12. The SVM model already contains the full predictive signal with negligible accuracy loss (~0–2%), is faster, and ensures reproducibility across all systems.

Troubleshooting

FileNotFoundError: data/pico/P_model.npz

You likely skipped the Git LFS step. Run git lfs pull to download the actual model weights (only required for app.py and app1.py).

ModuleNotFoundError: No module named 'rct_reviewer'

Ensure you are running the command from the root directory, or set your PYTHONPATH:
export PYTHONPATH=$PYTHONPATH:$(pwd)

Can't find model 'en_core_web_sm'

You forgot to download the spaCy model. Run:
python -m spacy download en_core_web_sm

Frequently Asked Questions

What is RCT-Reviewer and how does it relate to RobotReviewer?

RCT-Reviewer is a modernized, standalone version of the original RobotReviewer developed by Marshall, Kuiper, and Wallace. It preserves the original machine learning models trained on 12,808 randomized controlled trial PDFs while modernizing the infrastructure with Streamlit, PyMuPDF, and Python 3.12 compatibility. It is designed as a third-party tiebreaker reference tool for systematic reviews.

How accurate is RCT-Reviewer for Risk of Bias assessment?

RCT-Reviewer achieves 71.0% accuracy for Risk of Bias judgments, performing within 8% of human expert consensus (which stands at 78.3%). In a randomized Cochrane user trial, the models demonstrated 87% precision and 90% recall for identifying the exact text snippets supporting each bias judgment.

Why was the CNN model removed in favor of an SVM-only pipeline?

CNN models were removed because they depend on TensorFlow/Keras .h5 files which break on Python 3.11 and 3.12. The Linear SVM model already contains the full predictive signal with negligible accuracy loss of 0 to 2 percent, is faster, and ensures reproducibility across all systems.

Do I need Java or GROBID to run RCT-Reviewer?

No. Unlike the original RobotReviewer which required GROBID and Java, RCT-Reviewer uses PyMuPDF (fitz) for native Python PDF parsing and spaCy for sentence segmentation. This eliminates the Java and Docker dependencies entirely.

What are the different ways to run RCT-Reviewer?

There are three modes. app.py uses compressed .joblib and .npz files for modernized local storage. app1.py uses legacy .pickle, .pck, and .npz files. app2.py connects directly to the Hugging Face Hub repository Aurumz/RCT-Reviewer without requiring large local files. The recommended mode is app2.py for the latest version.

Is RCT-Reviewer free and open source?

Yes, RCT-Reviewer is completely free and open source under the GNU General Public License v3.0. It is a derivative work of RobotReviewer and the source code is available on GitHub.

Can RCT-Reviewer replace human reviewers in systematic reviews?

No. RCT-Reviewer is designed as a third-party tiebreaker reference. Standard systematic review guidelines require two independent human reviewers. When they disagree, RCT-Reviewer provides an instant, objective, data-driven third opinion to help resolve ties. Real-world feasibility studies show human reviewers accept the tool's judgments at a rate equal to that of their human peers.

Where can I download the RCT-Reviewer model files?

The default RCT-Reviewer model files in .joblib format are available on Hugging Face at huggingface.co/Aurumz/RCT-Reviewer. The legacy .pickle and .pck model files are available at huggingface.co/Aurumz/RCT-Reviewer-pickle. For running without large downloads, use app2.py which fetches models directly from the Hugging Face Hub.

Is my uploaded PDF data safe?

If you self-host RCT-Reviewer locally, all processing happens entirely on your machine — no data leaves your system. The hosted online version at rct-reviewer.streamlit.app processes PDFs in-memory and does not permanently store uploaded files.
